belfoot_img - сервис, предназначенный для работы с изображениями с целью их сжатия для последующего хранения.
Техническое задание:
Разработка сервиса, обрабатывающего входящие запросы, содержащие гиперссылки на изображения, с целью их сжатия до 
файла формата .jpeg размером, не превышающим 180кб с сохранением пропорций изображения и минимальной потерей качества.
Сервис должен хранить данные в СУБД PostgreSQL. Доступ к API извне должен быть закрыт. Реализовать возможность получения
данных по передаваемому в POST-запросе id статьи через GET-запрос

Реализация:
1. Сервис состоит из двух приложений, реализованных по RESTFul архитектуре:
1.1 admin_auth - реализация закрытого API на основе базовой аутентификации(login:password). Данные должны передаваться 
в BODY запроса в формате JSON вместе с данными об изображении {"user": {"username": "имя_пользователя", "password": "пароль"}}.
Приложение реализовано без создания клиентских сессий для избежания хранения данных в куки из-за отсутствия на данный
момент функционала по работе с куки-файлами на платформе CREMAX. Данные о пользователе должны передаваться в каждом запросе.
1.2 img_converter - непосредственно работа с изображением. После обработки через admin_auth запрос перенаправляется в img_converter
в формате {"data": {"url": "ссылка_на_изображение", "article_id": "идентификатор_статьи"}}. Изображение проверяется на
соответствие условиям, указанным в ТЗ и в случае, если условия не соблюдены, приложение высчитывает коэффициент соотношения
сторон, пользуясь которым уменьшает разрешение изображения пропорционально. Далее изображение проходит очередную проверку
на соответствие условиям ТЗ, и в случае, если условия не соблюдены, изображение сжимается по качеству с шагом в 5% до достижения
требуемого объёма памяти. Изображение сохраняется в базу данных в формате base64 string.
2. Полный шаблон запроса выглядит следующим образом:
{
    "user": {        
        "username": "имя_пользователя",
        "password": "пароль_пользователя"
    },
    "data": {
        "url": "ссылка_на_изображение",
        "article_id": "идентификатор_изображения"
    }
}

3. Перед деплоем настроить базу данных используя СУБД PostgreSQL
4. Для запуска сервера подразумевается использование gunicorn. Используемый reverse-proxy - NGINX
